SQL AI Copilot - Windows 打包使用指南
========================================

快速开始：
----------
1. 双击运行 "build_and_package.bat"
2. 选择选项 4 "完整打包（推荐）"
3. 等待打包完成
4. 在生成的 "SQL_AI_Copilot_Windows" 文件夹中找到可执行文件

详细步骤：
----------
1. 环境检查
   - 运行 build_and_package.bat
   - 选择选项 1 检查环境依赖
   - 确保 g++ 编译器和 libcurl 库可用

2. 安装依赖（如需要）
   - 如果缺少编译器，选择选项 2
   - 按照提示安装 MinGW-w64

3. 编译打包
   - 选择选项 4 进行完整打包
   - 等待编译和打包完成

4. 使用程序
   - 进入 SQL_AI_Copilot_Windows 文件夹
   - 双击 "启动程序.bat"
   - 按照程序提示输入 SQL 和执行计划

文件说明：
----------
- build_and_package.bat：主构建工具
- package_windows.bat：完整打包脚本
- install_dependencies.bat：依赖安装脚本
- build_windows.bat：简单编译脚本

输出文件：
----------
SQL_AI_Copilot_Windows/
├── sql_ai_copilot.exe     # 主程序
├── 启动程序.bat           # 推荐启动方式
├── 配置向导.bat           # 配置帮助
├── 使用说明.txt           # 详细说明
├── 版本信息.txt           # 版本信息
├── config/
│   └── ai_models.json     # AI配置文件
├── include/               # 头文件（参考用）
└── docs/                  # 文档文件夹

故障排除：
----------
1. 编译失败
   - 检查是否安装了 g++ 编译器
   - 确保 libcurl 库可用
   - 检查源代码文件是否完整

2. 程序运行失败
   - 检查网络连接
   - 验证 API 密钥配置
   - 确认配置文件格式正确

3. 依赖问题
   - 使用 install_dependencies.bat 安装依赖
   - 或手动安装 MinGW-w64

技术支持：
----------
如遇到问题，请提供：
- 错误信息
- 操作系统版本
- 编译器版本
- 网络连接状态

注意事项：
----------
- 需要网络连接调用 AI API
- 确保 API 密钥有效且有足够配额
- 建议使用启动脚本而非直接运行 exe
- 首次使用请检查配置文件
